# nokia\_read\_novel：将网页内容转换为诺基亚浏览器快照格式

## 项目背景

### 一个程序员的"摸鱼"需求

故事要从2026年说起。是的，您没看错，在2026年，我仍然在使用一部诺基亚功能机——具体来说是 Nokia 215 4G。这部手机成为了我上课时的"秘密武器"，因为它足够简单，不会像智能手机那样让我分心，但又能满足基本的通讯需求。

然而，作为一个热爱阅读的人，我总觉得少了点什么——我想在上课的时候看小说。

直接用诺基亚浏览器访问小说网站？试过了，要么是网络问题加载不出来，要么是设备缓存限制导致页面无法加载。现代网站的复杂HTML和CSS对这部老设备来说实在是太沉重了。

于是，一个想法诞生了：**能不能把小说内容直接"硬塞"进浏览器里？**

### 逆向工程的起点

我发现诺基亚浏览器有一种"快照"功能，可以保存网页内容供离线阅读。这个快照文件不是标准的HTML，而是一种专有格式。通过对多个快照文件的二进制分析，我终于揭开了这种格式的神秘面纱。

`nokia_read_novel` 就是这样诞生的——一个能将标准HTML或纯文本内容转换为诺基亚浏览器快照格式的工具。

### 为什么选择纯文本

在实际使用中，我发现**纯文本是最佳选择**。HTML格式虽然支持丰富的样式，但会占用大量空间，导致单个快照文件能容纳的小说内容有限。而纯文本格式可以在有限的存储空间内塞进更多文字，让阅读体验更加流畅。

当然，纯文本模式也有其局限性：默认的纯文本转换模式不带有小说导航，因此翻页可能要翻很久。这是因为小说文本的标题形式千奇百怪，难以自动识别和提取。因此，对于较长的小说，**建议拆分成多个部分**分别转换，这样不仅便于阅读，还能避免设备内存限制带来的内容截断问题。

## 技术原理

### 格式逆向分析

通过对诺基亚浏览器快照文件的深入逆向分析，我发现这种专有格式由以下几个部分组成：

| 部分     | 字节范围   | 内容  | 说明               |
| ------ | ------ | --- | ---------------- |
| 头部长度   | 0-3    | 4字节 | 小端序，标识头部总长度      |
| 版本号    | 4-7    | 4字节 | 小端序，当前为1         |
| 保留字段   | 8-11   | 4字节 | 固定为0             |
| URL长度  | 12-15  | 4字节 | 小端序              |
| 神秘字段长度 | 16-19  | 4字节 | 小端序              |
| URL标志  | 20-23  | 4字节 | 固定为"http"的ASCII码 |
| URL数据  | 24+    | 变长  | UTF-8编码的URL      |
| 神秘字段   | 紧随URL后 | 变长  | 可填充任意数据          |
| HTML内容 | 末尾     | 变长  | 标准HTML内容         |

### 核心技术点

1. **二进制格式处理**：使用Python的`struct`模块处理小端序字节流
2. **编码转换**：确保URL和HTML内容正确使用UTF-8编码
3. **灵活配置**：支持自定义URL和神秘字段长度

## 功能特性

- **多格式支持**：支持HTML文件和纯文本文件输入
- **自动转换**：纯文本文件自动包装为HTML格式，包含基础样式
- **URL伪装**：支持自定义伪装URL地址，便于在浏览器中识别
- **文件优化**：神秘字段可配置长度，支持设置为0以最小化文件大小
- **命令行界面**：简洁易用的命令行参数设计

## 使用方法

### 基本转换

```bash
# 转换HTML文件
python3 nokia_snapshot_converter.py input.html output.html --url "http://example.com"

# 转换纯文本文件
python3 nokia_snapshot_converter.py input.txt output.html --url "http://example.com" --type text

# 设置最小文件大小（神秘字段长度为0）
python3 nokia_snapshot_converter.py input.html output.html --url "http://example.com" --mystery 0
```

### 参数说明

| 参数        | 缩写 | 说明                | 默认值                  |
| --------- | -- | ----------------- | -------------------- |
| input     | -  | 输入文件路径（必需）        | -                    |
| output    | -  | 输出文件路径（必需）        | -                    |
| --url     | -u | 伪装的URL地址          | <http://example.com> |
| --type    | -t | 输入文件类型（html/text） | html                 |
| --mystery | -m | 神秘字段长度            | 14                   |

### 使用场景

**场景一：小说阅读**

```bash
python3 nokia_snapshot_converter.py novel.html novel_snapshot.html --url "http://www.novel-site.com"
```

**场景二：轻量阅读**

```bash
python3 nokia_snapshot_converter.py readme.txt readme_snapshot.html --type text --mystery 0
```

## 神秘字段探索

通过大量实验验证，神秘字段具有以下特点：

- **非必需性**：可以设置为0长度，不影响文件正常显示
- **内容无关**：可以填充任意值（建议使用全0）
- **用途推测**：可能是浏览器内部用于跟踪或缓存管理的字段

## 注意事项

1. 输出文件建议使用 `.html` 扩展名
2. 纯文本文件会自动添加基础HTML结构和样式
3. 伪装URL仅用于浏览器内部标识，不影响实际内容显示
4. 建议在实际设备上测试转换后的文件兼容性
5. **加载时间**：如果塞入的内容足够多，加载时间可能会延长到几十秒。如果期间手机自动熄屏，只需按亮屏幕继续等待即可
6. **内容限制**：小说靠后的内容可能无法阅读，这是设备内存限制导致的。不过长篇小说仍能阅读大部分内容，短篇甚至可以全部塞进去
7. **导航体验**：默认的纯文本转换模式不带有小说导航功能，因此翻页可能需要翻很久。由于小说文本的标题形式千奇百怪，难以自动识别并生成章节导航。建议将长篇小说拆分成多个部分分别转换，这样不仅便于阅读定位，还能避免单文件过大导致的加载问题
